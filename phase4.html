// components/DepthShell.tsx // ══════════════════════════════════════════════════════════ // Context provider + CSS variable injector. // Wrap the hole page with <DepthShell depth={currentLevel}>. // Child components call useDepth() to get the current theme. // CSS variables are applied to the shell div so all // downstream CSS can reference --d-* without prop-drilling. // ══════════════════════════════════════════════════════════ 'use client' import { createContext, useContext, useEffect, useRef } from 'react' import { getDepthTheme, depthToCSSVars, type DepthTheme } from '@/lib/depth-theme' interface DepthCtx { theme: DepthTheme; depth: number } const DepthContext = createContext<DepthCtx>({ theme: getDepthTheme(1), depth: 1, }) export function useDepth() { return useContext(DepthContext) } interface Props { depth: number; children: React.ReactNode } export default function DepthShell({ depth, children }: Props) { const theme = getDepthTheme(depth) const ref = useRef<HTMLDivElement>(null) // Inject CSS custom properties onto the shell root element. useEffect(() => { const el = ref.current if (!el) return const vars = depthToCSSVars(theme) Object.entries(vars).forEach(([k, v]) => el.style.setProperty(k, v)) }, [theme]) return ( <DepthContext.Provider value={{ theme, depth }}> <div ref={ref} className="depth-shell" data-depth={depth} // components/AtmosphericBg.tsx // ══════════════════════════════════════════════════════════ // Canvas-based atmospheric background. // Renders slow, animated radial blobs tinted with the depth // accent colour. Blob count = depth level. FPS intentionally // capped at 12 — the stuttery feel adds to the degradation. // Lives at z-index:0, behind all content. // ══════════════════════════════════════════════════════════ 'use client' import { useEffect, useRef } from 'react' import { useDepth } from './DepthShell' const FPS = 12 // low by design — degraded-signal aesthetic export default function AtmosphericBg() { const { theme, depth } = useDepth() const canvasRef = useRef<HTMLCanvasElement>(null) const rafRef = useRef<number>(0) const lastRef = useRef<number>(0) useEffect(() => { const canvas = canvasRef.current if (!canvas) return const ctx = canvas.getContext('2d')! const resize = () => { canvas.width = window.innerWidth canvas.height = window.innerHeight } resize() window.addEventListener('resize', resize) let t = 0 const draw = (now: number) => { rafRef.current = requestAnimationFrame(draw) if (now - lastRef.current < 1000 / FPS) return lastRef.current = now t += 0.004 const { width: w, height: h } = canvas ctx.fillStyle = theme.bg ctx.fillRect(0, 0, w, h) // One animated blob per depth level, orbit the canvas centre for (let i = 0; i < depth; i++) { const angle = t + (i * Math.PI * 2) / depth const r = Math.min(w, h) * (0.28 + 0.08 * Math.sin(t * 0.7 + i)) const x = w / 2 + Math.cos(angle) * r * 0.5 const y = h / 2 + Math.sin(angle) * r * 0.4 const grd = ctx.createRadialGradient(x, y, 0, x, y, r) grd.addColorStop(0, theme.accent + '20') grd.addColorStop(1, 'transparent') ctx.fillStyle = grd ctx.beginPath() ctx.arc(x, y, r, 0, Math.PI * 2) ctx.fill() } } rafRef.current = requestAnimationFrame(draw) return () => { cancelAnimationFrame(rafRef.current) window.removeEventListener('resize', resize) } }, [theme, depth]) return ( <canvas ref={canvasRef} aria-hidden style={{ position: 'fixed', inset: 0, zIndex: 0, pointerEvents: 'none', // Opacity climbs with depth — deeper = more atmospheric opacity: 0.55 + depth * 0.06, transition: 'opacity 1200ms ease', }} /> ) }

Design Notes depth-theme.ts
The Depth Theme System

Seven depth tiers, each a complete design token set. The key insight: colour alone isn't enough. The visual identity at each level is a combination of palette shift, grain opacity, vignette strength, typography weight, and scanline presence.

#NameAccentNoiseGlitch
1Surfaceviolet2%off
2Shallowindigo5%off
3Murkycyan9%30s
4Deepteal14%15s
5Abyssamber20%8s
6Voidred28%4s
7Bottomblood38%2s
The accent colour arc is intentional: violet (safe, familiar) → cyan (strange) → amber (warning) → red (danger). Users feel the shift even before they consciously notice it.
✓ depthToCSSVars() converts a theme object to CSS custom properties. This is the only bridge between TypeScript config and CSS — one call, 12 variables, everything downstream uses var(--d-*).
Depth is clamped to 1–7 by getDepthTheme(). A thread with 12 levels uses tiers 1, 2, 3… then the final levels all share tier 7. The bottom is the bottom.
Depth-Aware Global CSS

The CSS file defines structure — the body, utility classes, keyframes — but never hardcodes colours. Every colour reference is var(--d-*), which DepthShell swaps at runtime.

The .pip strip encodes all solve state in CSS: .solved, .current, .locked. The glow on .current uses pip-pulse — an alternating keyframe that breathes with the page's ambient atmosphere.
The .locked-content class uses filter: blur(var(--d-blur)). At depth 7 that's 40px — text is completely illegible. This is the "spoiler gate" for level content a user hasn't unlocked yet.
✓ The shake animation uses translateX only, not margin. This keeps the layout stable (no reflow) and runs on the compositor thread for guaranteed 60fps even on wrong answers.
DepthShell — Context + Injector

The shell does two things: provides a React context so child components can read the current theme without prop-drilling, and injects CSS custom properties directly on the DOM node so CSS rules work without JSX logic.

Why not put the CSS vars on :root? Because a user could have two holes open in different tabs — or you might render a preview alongside live content. Scoping to the shell div means the variables don't leak globally.
The three atmospheric overlays — noise, scanlines, vignette — are position:fixed with pointer-events:none. They cover the entire viewport but never intercept clicks, scrolls, or keyboard events.
⚠️ Scanlines only activate at depth ≥ 4. Adding them at depth 1 would immediately signal "something is wrong" — and at the surface, nothing is wrong yet.
✓ The vignette hex alpha is computed from vignetteStr: Math.round(strength * 255).toString(16). At depth 7, strength = 0.95 → alpha = f2 — the edges are nearly black.
AtmosphericBg — Canvas Background

A 2D canvas renders slowly orbiting radial blobs tinted with the depth accent colour. The blob count equals the depth level — at depth 7 you have 7 overlapping blobs filling the void.

FPS is deliberately capped at 12 via lastRef timestamp gating. At 60fps the background would feel like a normal animation. At 12fps it feels like degraded footage — a signal fighting to come through.
The canvas sits at z-index:0, below the noise overlay (z:1), scanlines (z:2), and vignette (z:3). All content is at z-index 4+. The layers stack: canvas → noise → scanlines → vignette → content.
✓ cancelAnimationFrame is called in the useEffect cleanup. Without this, every time depth changes (which re-runs the effect), you'd leak a running animation loop.
LevelTransition — The Solve Moment

The transition is the payoff. Three timed phases: in (fade up from 0 opacity), hold (show XP + message, 60% of total duration), out (fade away, onDone fires).

Duration scales with depth via theme.transitionMs. At depth 1 it's 600ms — crisp. At depth 7 it's 2600ms — a slow, deliberate sinking. The descent itself feels heavier the deeper you go.
The XP number uses a spring-in animation (translateY(20px) → translateY(0)) that fires 80ms after the overlay becomes visible. This slight delay makes it feel like the number "arrives" rather than just appearing.
✓ GlitchText is used for the atmospheric message. At depth 7 the message "There is no coming back." glitches continuously during the hold phase — fitting punctuation for the bottom of the hole.
PuzzleInput — The Answer Box

Every colour state (border, glow) pulls from the depth theme. At depth 1, a correct answer glows violet. At depth 5, it glows amber. The input's feedback palette shifts with the environment.

The shake animation uses the Web Animations API (element.animate()) rather than a CSS class toggle. This avoids the flash-of-wrong-class that can occur when React re-renders during the animation, and gives precise programmatic control.
⚠️ A locked input shows '██████████' as placeholder. Combined with filter:blur() on the input itself, this communicates "sealed" without needing a separate locked-state component.
Attempt count is tracked server-side (via the useAttempt hook hitting the API). The UI uses remaining from the hook to drive the counter display — never trust the client count alone.
DepthMeter — The Descent Gauge

A vertical fill bar with dot indicators per level. The fill uses a spring cubic-bezier (0.34,1.56,0.64,1) so it slightly overshoots on descent — a tiny bounce that makes progress feel physical.

The depth tier name is rendered vertically via writing-mode:vertical-rl + rotate(180deg). It updates with the theme transition so "Surface" fades to "Murky" smoothly as the depth variable changes.
✓ The component is purely presentational — it reads from useDepth() and props only. No API calls, no side effects. Safe to render anywhere without worrying about data fetching.
GlitchText — Degraded Signal

Three layers: a red-offset clone (back), the real text (middle), an accent-coloured clone (front). The clones are opacity:0 at rest. The interval fires glitch-h on one and glitch-v on the other, then resets after 320ms.

The interval is jittered: glitchRate ± 40%. A perfectly regular glitch reads as intentional — a designed animation. Random intervals make it feel like signal corruption, which is the point.
The as prop defaults to 'span' but accepts any HTML tag. Use <GlitchText as="h1"> for the thread title to apply the effect to a block-level element without breaking semantic HTML.
✓ Both clone spans have aria-hidden. Screen readers see only the main text layer — the glitch is purely cosmetic and doesn't affect accessibility.
